# NORIKAERU Roadmap

## Phase 1: UI Lock

- ルート選択、ルートキープ、ルート変更、乗換完了、到着までの基本フローを安定させる。
- 表示順、余白、アウトライン、文言などの細かい違和感を潰す。
- スマホ画面で通勤中に迷わない操作だけを残す。

## Phase 2: Data Model

- `app/data/` に平日ダイヤJSONを追加する。
- あびこ、動物園前、淡路の時刻表データを分けて管理する。
- 天六行きなど、淡路まで行かない列車を除外するルールをJSON生成時に反映する。

## Phase 3: Route Engine

- 現在時刻から「一本前」「現在対象」「一本後」を抽出する。
- あびこ発、動物園前着、動物園前発、淡路着を組み合わせる。
- 会社到着（予定）は淡路着 + 17分で自動計算する。

## Phase 4: UI Connect

- 固定デモデータをRoute Engineの出力に差し替える。
- 白/グレータイルを実ダイヤから生成する。
- 想定外の乗換も記録できるようにする。

## Phase 5: Records

- 到着時に当日の通勤記録を保存する。
- サマリー画面で日付ごとの記録を見られるようにする。
- 成功/想定外/未選択などの状態を後から見返せる形にする。

## Phase 6: Settings

- 出勤/退勤モードの設定を実装する。
- 会社到着加算分、対象駅、表示本数などを変更できるようにする。
- 将来的に休日ダイヤや帰宅ルートを追加できる構成にする。
